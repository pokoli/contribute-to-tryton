Before contributing code

  1. Search existing discussions/issues
    • On the discourse server:
      https://discuss.tryton.org/
    • On the bug tracker:
      https://bugs.tryton.org
  2. Ask/Discuss on the IRC/ML
    • irc://irc.freenode.net/tryton
    • http://groups.tryton.org
  3. Read contribution rules
    4. Patches always on latest development version
  4. Read code style
    5. PEP8 (with exceptions), 80cols
  5. Inform about your plans
    6. Assign to issue yourself & state=in progress

Submiting code

  1. Clone: hg clone https://hg.tryton.org/trytond
  2. Modify
    3. vi filename.py
  3. Run tests
    4. python setup.py test
  4. Upload
    5. hg review -m "trytond: Explanation of your change"
  5. Review
    6. mutt & browser
  6. Re-Upload
    7. hg review -m "fix comments"
  7. Commit
    8. Normally by a core developer

Code Review

  • Before commit
  • Using rietveld:
    https://codereview.tryton.org

  • On Other users code
